# Mediator Sync Companion (Figma Plugin)

This plugin is the write-capable bridge for Mediator to apply named-node updates directly in Figma.

## Files

- manifest.json
- code.ts (source)
- code.js (runtime file loaded by Figma)
- ui.html
- ui.ts (source)
- ui.js (runtime file loaded by ui.html)

## Load In Figma

1. Open Figma desktop app.
2. Go to Plugins -> Development -> Import plugin from manifest.
3. Select [figma-plugin/manifest.json](manifest.json).
4. Run Plugins -> Development -> Mediator Sync Companion.

## Supported payload shape

```json
{
  "textBindings": {
    "panel_backend_layers": "Updated backend summary",
    "panel_dashboard_activity_feed": "Last sync: 6:42 PM",
    "status_global_healthy": "Healthy"
  },
  "createNodes": [
    {
      "type": "frame",
      "name": "panel_generated_architecture",
      "x": 120,
      "y": 180,
      "width": 720,
      "height": 380,
      "fillHex": "#10151f"
    },
    {
      "type": "text",
      "name": "status_generated_at",
      "x": 148,
      "y": 212,
      "width": 300,
      "height": 24,
      "text": "Generated by Mediator"
    }
  ]
}
```

The plugin also accepts the shortcut form where top-level `panel_`, `status_`, `log_`, `input_`, and `meta_` keys map directly to text values.

## Notes

- If a target node is not a text layer, the plugin tries the first descendant text node.
- For local push, you can host payload JSON on `http://127.0.0.1:8080/figma-plugin` and use the plugin UI `Fetch Endpoint` button.
- Figma cannot accept inbound arbitrary sockets from your desktop app directly; use pull (plugin fetches endpoint), file import, or webhook-assisted flow.
